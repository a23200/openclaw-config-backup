# 多 Agent 协作工作设计文档 V2

## Agent 1: 总控路由 Agent (Master Router)
- **角色定位**: 整个系统的入口和指挥官。它是用户的唯一直接交互对象。
- **核心任务**:
    1.  **接收指令**: 接收用户下达的任何高阶、模糊的指令。
    2.  **任务拆解**: 理解用户意图，将复杂任务拆解成一系列可以由其他专员处理的、具体的子任务。
    3.  **任务派发**: 根据子任务的性质，选择最合适的“专员 Agent”并将任务派发给它。
    4.  **监督与整合**: 监控各个专员 Agent 的工作进度，收集它们返回的结果，然后将结果整合成最终成品，交付给用户。
