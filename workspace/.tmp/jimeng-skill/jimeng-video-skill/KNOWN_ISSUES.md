## 4. check-result 误判历史报错为当前失败（需修复）

**问题**：`check-result` 只看页面上最新的一条错误，但即梦页面会保留所有历史生成记录（包含旧的 face_detected 报错）。当新任务还在生成中时，`check-result` 会把历史记录里的旧报错当成当前任务的失败。

**正确判断逻辑**：应该找到与当前 prompt 对应的那条记录，看该记录下面是否有报错，而不是看页面上任意一条报错。

**建议修复**：`check-result` 改为只检查最新提交的那条任务（ref 最大的那组），或者通过 prompt 文本匹配找到对应记录再判断状态。

## 5. 即梦 face_detected 误触发（动漫风格也会触发）

**问题**：即梦对 prompt 里描述的人物也会做人脸检测，即使是动漫/卡通风格也会触发 face_detected。

**结论**：face_detected 是生成结果的审核问题，不是上传素材的问题。动漫风格 prompt 描述人物外貌不会必然触发，需要实际生成后才知道。

**建议**：不要因为 face_detected 就修改 prompt，先确认是当前任务的报错还是历史记录的报错。
