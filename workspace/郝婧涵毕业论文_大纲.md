# 《基于 Spring Boot 与 Vue 的流浪宠物救助系统》论文大纲（增强版）

## 一、选题定位
- 题目：基于 Spring Boot 与 Vue 的流浪宠物救助系统的设计与实现
- 研究对象：龙子湖高校园区流浪动物救助与领养业务
- 研究目标：解决救助信息分散、领养流程不透明、救助站协同效率低的问题
- 技术路线：Spring Boot + Vue + MySQL + 前后端分离架构

## 二、论文总体结构建议

### 第1章 绪论
#### 1.1 研究背景
- 高校园区流浪动物数量增加
- 救助信息依赖微信群、朋友圈、人工登记，数据分散
- 救助、领养、募捐、审核流程缺少统一平台

#### 1.2 研究意义
- 理论意义：为校园公益救助信息化建设提供参考
- 实践意义：提升救助效率、透明度和规范化水平
- 社会意义：增强动物保护意识，促进文明校园建设

#### 1.3 国内外研究现状
- 国内：偏向平台宣传与基础登记，系统化、闭环型平台较少
- 国外：救助、领养、志愿者协作、社区联动机制较成熟
- 评述：现有研究在校园场景适配、流程闭环、角色协同方面仍有提升空间

#### 1.4 研究内容与目标
- 构建一套适用于高校场景的流浪宠物救助系统
- 实现用户端与管理员端的核心业务功能
- 完成数据库设计、系统实现与测试验证

#### 1.5 创新点
- 面向高校园区救助场景进行定制化设计
- 将动物信息、领养申请、募捐、论坛交流、救助站管理整合到统一平台
- 采用前后端分离架构提升可维护性与扩展性

### 第2章 相关技术与开发环境
#### 2.1 Java 与 Spring Boot
- Java 语言优势
- Spring Boot 自动配置、快速开发、便于 REST API 构建

#### 2.2 Vue 前端框架
- 组件化开发
- 响应式数据绑定
- 适合后台管理系统与用户门户构建

#### 2.3 MySQL 数据库
- 关系型数据管理
- 适合业务实体清晰、表关系明确的系统

#### 2.4 B/S 架构
- 浏览器访问，部署维护成本低
- 适合校园多角色、多终端访问场景

#### 2.5 开发与运行环境
- 后端：JDK 17 / Spring Boot 3.x / Maven
- 前端：Node.js / Vue 3 / Vite / Element Plus
- 数据库：MySQL 8.x
- 开发工具：IntelliJ IDEA、VS Code、Navicat/Postman
