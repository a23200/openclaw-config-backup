## 2. 通用人工操作原则

适用于上面所有项目：

1. 先确认自己操作的是“正式项目”还是“测试/临时目录”
2. 先确认当前端口、运行实例、浏览器页面是不是同一个
3. 尽量优先使用项目内已有说明文档，不要凭记忆乱操作
4. 对外发布、删除、覆盖、批量导入等动作，视为高风险动作，人工确认后再执行
5. 能用本地配置文件/说明文档驱动的，就不要先改代码
6. 出现问题先分清：
   - 是前端缓存问题
   - 是后端服务问题
   - 是外部依赖问题
   - 还是账号/登录态问题

## 3. 各项目的人为使用方式与 SOP

# 3.1 PPT 项目 SOP

## 用途
用于生成、编辑、导出 PPT 或相关展示材料。

## 人为使用方式
1. 进入项目目录
2. 读取项目 README / 文档
3. 准备输入内容（主题、结构、素材、图表、文案）
4. 执行生成或导出流程
5. 检查输出文件是否完整
6. 人工复核版式、中文排版、图片显示、导出结果

## 标准 SOP
1. 明确本次 PPT 的目标（汇报 / 提案 / 样张 / 复盘）
2. 准备原始内容
3. 先小范围验证生成链路
4. 再批量生成
5. 输出后人工检查
6. 将最终版本单独归档

## 人工重点检查项
- 中文排版是否错位
- 图表是否缺失
- 封面和目录是否正常
- 导出文件是否能打开

---

# 3.2 App 项目 SOP

## 用途
用于独立 App 的开发、测试与结构设计。

## 人为使用方式
1. 明确要做的是：新功能、修复、重构还是原型
2. 先看项目文档和目录结构
3. 启动本地开发环境
4. 只改当前任务相关模块
5. 改完后本地验证
6. 确认不影响其他功能

## 标准 SOP
1. 定义功能目标
2. 查看已有架构
3. 只做最小改动
4. 本地运行验证
5. 记录改动点

## 人工重点检查项
- 是否改坏原有流程
- 是否出现端口冲突
- 是否依赖了本机绝对路径

---

# 3.3 控制台 / Console 项目 SOP

## 用途
用作控制台、调度台或 OpenClaw 相关交互入口。

## 人为使用方式
1. 确认当前运行实例
2. 明确当前端口与配置
3. 观察控制台显示是否与实际状态一致
4. 对高风险动作先看权限和开关状态

## 标准 SOP
1. 先确认环境
2. 再确认展示数据来源
3. 再做页面/逻辑检查
4. 最后再考虑功能修改

## 人工重点检查项
- 是否展示旧缓存数据
- 是否误连到错误环境
- 是否把临时调试实例当成正式实例

## 人为使用方式（UI 方式）
1. 打开 `http://127.0.0.1:8787`
2. 在 AI 助手区输入题目，或点击热榜/随机抽题
3. 检查生成的标题、首帧、尾帧、视频提示词是否合理

## 人为使用方式（非 UI 方式）
1. 直接理解工作流文档
2. 按输入参数组织任务
3. 生成最终视频后准备发布配置
4. 直接走发布链路而不依赖 UI 页面

- `docs/WORKFLOW.md`
- `docs/DOUYIN_PUBLISH_FLOW.md`
- `docs/HEADLESS_CALL_FLOW.md`

## 标准 SOP
1. 确认打开的是 `8787`
2. 确认热榜来源可用
3. 生成题目与提示词
4. 提交即梦任务
5. 等待首帧 / 尾帧 / 视频完成
6. 检查后处理结果
7. 若需要，进入抖音发布链路
8. 保存最终视频与相关配置

## 人工重点检查项
- 端口是否正确
- 是否是旧前端缓存
- 热榜是否为空、来源是否正常
- 最终成片是 `final.mp4` 还是 `final_mastered.mp4`
- 硬字幕是否成功
- 发布脚本是否可用
- 不要混淆两种抖音发布模式：
  - 固定已登录 profile 的真实 Chrome 模式（历史已验证）
  - attach 当前桌面浏览器模式（用户当前想要，但本轮仍在排障）

## 给开发工具的额外提示
1. 优先阅读：
   - `docs/DOUYIN_PUBLISH_FLOW.md`
   - `docs/HEADLESS_CALL_FLOW.md`
2. 当前更稳的恢复策略是：先恢复 `persistent-profile` 发布模式，再单独修 `attach-current-browser` 模式
3. 不要把 `connectOverCDP()` 直接塞进原 `launchPersistentContext(...)` 逻辑里当作“已经支持 attach”

## 4. 与开发工具协作时的推荐方式

当要让专业开发工具接手时，建议遵守以下方式：

1. 先给出项目路径
2. 明确当前正式端口
3. 指定本次目标是：
   - 看代码
   - 改功能
   - 修 bug
   - 写文档
   - 跑链路
4. 如果有固定 SOP 或现有文档，先让它读取：
   - `PROJECTS_SOP.md`
   - 对应项目下 `docs/*.md`
5. 涉及发布、外部账号、生产数据时，明确说明是否允许真实执行
