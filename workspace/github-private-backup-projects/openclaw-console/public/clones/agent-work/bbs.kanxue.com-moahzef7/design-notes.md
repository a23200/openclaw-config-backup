# bbs.kanxue.com 风格提炼与设计说明

## 目标
不是复制原站源码或资源，而是提炼其视觉语言，做一个“像同一位设计师做的”全新静态页面。

## 观察到的关键视觉特征

### 1. 整体布局
- 采用典型的 `主内容 + 右侧边栏` 双栏结构。
- 页面主体宽度较克制，内容集中在中间，左右留白较多。
- 顶部是简洁细长的导航条，下面直接进入内容区。
- 内容按“频道/版块”分段，每段内部再用两列卡片排布。

### 2. 配色语言
- **主色**：偏亮的安全蓝 / 青蓝色，用在 Logo、导航高亮、按钮、图标强调上。
- **辅色**：橙色，用于主题数、次级强调信息，形成轻微对比。
- **背景色**：很浅的蓝灰白，给人干净、理性、偏技术社区的感觉。
- **文字色**：主文字偏深蓝灰，不使用纯黑，显得更现代、更柔和。
- **边框色**：极浅灰蓝色，弱化边界感，保证信息密度高但不刺眼。

### 3. 字体层级
- 标题较克制，不夸张，主要靠字重而不是大字号建立层级。
- 正文说明和辅助信息字号偏小，强化论坛/信息站的高密度感。
- 统计数字在右侧边栏中更醒目，形成仪表盘感。

### 4. 间距韵律
- 卡片内边距普遍不大，信息密度高但仍保持有序。
- 模块之间有明确段落间距，保证页面扫描效率。
- 列表项之间采用轻微 hover 和极细边界处理，视觉很轻。

### 5. 组件风格
- 卡片是轻边框 + 小圆角 + 很弱阴影，不做厚重拟物。
- 图标区通常是浅蓝底的小块或圆角容器，增强“工具/技术站”识别感。
- 按钮偏扁平，主按钮使用蓝色渐变，次按钮用白底描边。
- 右侧信息盒子类似运营后台/社区边栏卡片，偏数据面板风格。

## 本次实现中的设计决策

### 保留的“神韵”
- 保留了双栏框架和论坛频道式分区。
- 保留了蓝白主调、橙色辅助点缀、浅边框和高信息密度。
- 保留了“最新 / 热门 / 精选 + 频道矩阵 + 侧边排行”的结构节奏。
- 保留了技术社区的理性、轻盈、工具感，而不是做成营销页或博客页。

### 有意做出的原创变化
- Logo、文案、栏目名称和卡片结构均为重新组织，不直接复刻原站内容。
- 所有样式均为重新书写，没有抓取对方 CSS/JS/图片资源。
- 图标使用文字化几何占位方式，不使用原站素材。
- 页面做得更统一、更整洁，弱化了部分旧式论坛的碎片化感。

## 结论
这个版本追求的是：
- 一眼能看出参考了“安全技术论坛”的视觉语气；
- 但结构、文案、样式实现都属于全新组织；
- 更适合作为“风格仿制”而不是“源码镜像”。
