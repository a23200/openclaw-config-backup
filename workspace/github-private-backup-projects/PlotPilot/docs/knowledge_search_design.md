# 全书知识检索设计方案

## 当前问题
1. 检索结果展示简单，只有类型标签和文本
2. 没有分类筛选（人物、地点、事件、章节等）
3. 没有关联推荐（点击人物后显示相关章节、关系等）
4. 没有可视化预览（关系图缩略图、时间线等）

## 优化方案

### 1. 分类检索标签页

```
检索面板
├── 全部 (当前的通用检索)
├── 人物 (从三元组 entity_type='character' 筛选)
├── 地点 (从三元组 entity_type='location' 筛选)
├── 事件 (从三元组 entity_type='event' 筛选)
├── 章节 (从 chapter_summary 检索)
└── 概念 (从三元组 entity_type='concept' 筛选)
```

### 2. 检索结果增强

#### 人物卡片
```
┌─────────────────────────────────┐
│ 👤 林晨 [主角]                   │
│ 描述：天赋异禀的修真者...        │
│ 首次出现：第1章                  │
│ 相关章节：1, 2, 3, 5, 7          │
│ 关系：师徒→柳月, 朋友→张三       │
│ [查看详情] [查看关系图]          │
└─────────────────────────────────┘
```

#### 地点卡片
```
┌─────────────────────────────────┐
│ 🏙️ 江城 [核心地点]               │
│ 类型：城市                       │
│ 描述：修真世界的繁华都市...      │
│ 首次出现：第1章                  │
│ 相关章节：1, 2, 3                │
│ 属性：规模-大型城市, 功能-贸易中心│
│ [查看详情] [查看地图]            │
└─────────────────────────────────┘
```

#### 章节卡片
```
┌─────────────────────────────────┐
│ 📖 第1章：初入江城               │
│ 字数：3500字                     │
│ 摘要：主角林晨初到江城...        │
│ 关键人物：林晨, 柳月             │
│ 关键地点：江城, 天玄宗           │
│ [跳转章节] [查看大纲]            │
└─────────────────────────────────┘
```

### 3. 智能关联推荐

点击检索结果后，右侧显示关联信息：

```
选中：林晨
├── 基本信息
│   ├── 名称：林晨
│   ├── 类型：主角
│   └── 描述：...
├── 出场章节 (7章)
│   ├── 第1章：初入江城
│   ├── 第2章：...
│   └── ...
├── 人物关系 (3条)
│   ├── 师徒 → 柳月
│   ├── 朋友 → 张三
│   └── 敌对 → 王五
├── 相关事件 (5条)
│   ├── 拜师天玄宗
│   ├── 突破筑基期
│   └── ...
└── 关系图预览
    [显示以林晨为中心的关系网络]
```

### 4. 快速操作

- **引用到输入框**：将选中内容插入到写作区
- **添加到大纲**：将人物/地点添加到当前章节大纲
- **查看完整关系图**：跳转到人物/地点关系图页面
- **编辑三元组**：跳转到知识库编辑

### 5. 检索增强

#### 语义检索
- 支持模糊匹配："修真者" 能匹配到 "修炼者"、"修仙者"
- 支持同义词："主角" 能匹配到 "林晨"
- 支持关系查询："林晨的师父" 能找到 "柳月"

#### 高级筛选
```
检索框下方添加筛选器：
[重要程度: 全部 ▼] [章节范围: 全部 ▼] [标签: 全部 ▼]
```

#### 检索历史
```
最近检索：
- 林晨
- 江城
- 第1章
```

## 技术实现

### 后端 API
```python
# 分类检索
GET /api/v1/novels/{slug}/search?q=林晨&type=character&importance=core

# 关联查询
GET /api/v1/novels/{slug}/knowledge/{entity_id}/related
返回：{
  "chapters": [...],
  "relationships": [...],
  "events": [...]
}
```

## 用户体验流程

### 场景1：查找人物信息
1. 用户在检索框输入"林晨"
2. 系统返回人物卡片，显示基本信息、出场章节、关系网络
3. 用户点击"查看关系图"，跳转到人物关系图页面
4. 用户点击"引用到输入框"，将人物描述插入到写作区

### 场景2：查找章节内容
1. 用户在检索框输入"第1章"或"初入江城"
2. 系统返回章节卡片，显示摘要、关键人物、关键地点
3. 用户点击"跳转章节"，左侧章节列表自动选中第1章
4. 用户点击"查看大纲"，右侧切换到"分章叙事"标签

### 场景3：查找地点信息
1. 用户在检索框输入"江城"
2. 系统返回地点卡片，显示类型、描述、相关章节、属性
3. 用户点击"查看地图"，跳转到地点关系图页面
4. 用户点击地点节点，右侧显示详细信息

## 数据流设计

```
用户输入 "林晨"
    ↓
前端发送检索请求
    ↓
后端查询三元组表 (entity_type='character', subject='林晨')
    ↓
后端查询章节表 (key_events LIKE '%林晨%')
    ↓
后端查询关系表 (subject='林晨' OR object='林晨')
    ↓
后端聚合数据返回
    ↓
前端渲染人物卡片
    ↓
用户点击"查看关系图"
    ↓
前端跳转到 /book/{slug}/characters
    ↓
关系图页面高亮显示"林晨"节点
```

## 与现有功能的整合

### 与三元组的关系
- 检索结果来自三元组数据
- 点击"编辑"跳转到知识库编辑三元组
- 三元组更新后，检索结果自动更新

### 与关系图的关系
- 检索结果可以跳转到关系图
- 关系图可以跳转回检索（点击节点后显示检索结果）
- 双向导航，无缝切换

### 与章节的关系
- 检索结果可以跳转到章节
- 章节编辑时可以引用检索结果
- 章节保存后，检索结果自动更新

## 总结

这个设计方案的核心思想是：
1. **统一数据源**：所有检索都基于三元组数据
2. **智能关联**：自动推荐相关信息
3. **快速操作**：一键引用、跳转、编辑
4. **可视化**：关系图、时间线、卡片展示
5. **无缝集成**：与现有功能（三元组、关系图、章节）深度整合
