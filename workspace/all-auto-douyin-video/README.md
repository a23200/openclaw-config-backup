# 🎬 抖音全自动视频生成器 (All-Auto Douyin Video)

这是一个强大的自动化脚本工具。它能“读懂”你提供的结构化剧本，全自动完成：**分镜生图 -> 镜头动画 -> 智能配音 -> BGM混音 -> 最终视频合成**，一气呵成。

## ✨ 核心亮点
*   **对话式“剧本”输入**：告别复杂的参数配置，只需按照约定的格式写一段文字即可。
*   **一致性分镜**：采用“固定场景描述 + 镜头动作差分”的策略，确保 DALL-E 3 生成的连续画面风格统一。
*   **专业级视听组合**：自动将静态图片转化为带有呼吸感的动态视频；OpenAI TTS 媲美真人配音；智能动态降低背景音乐音量（Ducking），凸显人声。

---

## 🛠️ 第一步：环境与配置准备

在第一次使用前，你需要做一点小小的准备工作：

1.  **系统要求**：
    *   确保你的电脑安装了 **Python 3**。
    *   **非常重要**：必须安装了音视频处理神器 **FFmpeg**。（Mac 用户在终端执行 `brew install ffmpeg` 即可安装）。
2.  **填入 API Key**：
    *   用文本编辑器打开本项目根目录下的 `run.py` 文件。
    *   找到第 10 行：`OPENAI_API_KEY = "YOUR_OPENAI_API_KEY_HERE"`
    *   把你真实的 OpenAI API Key 替换掉引号里的内容。
    *   *(注：如果没有填写真实 Key，程序将运行在“模拟模式”，输出紫色的占位图片和无声配音，仅供测试流程使用)*。

## 🚀 第二步：如何制作视频？（操作过程）

操作非常简单，你只需要准备好“剧本”：

### 1. 准备剧本
请严格按照以下格式，在一个文本编辑器里写好你的内容（注意保留空行和标题）：

```text
固定场景/角色：
一只身穿红色宇航服的可爱小猫宇航员，它有圆圆的头盔和蓝色的眼睛。它正站在月球的灰色表面上，远方的天空中挂着一颗美丽的蓝色地球。整个画面的风格是皮克斯动画电影，光线柔和，细节丰富，电影感十足。

镜头描述：
1. 小猫宇航员刚刚从着陆器上跳下来，正好奇地环顾四周，小脑袋微微歪着。
2. 它发现了一块闪闪发光的蓝色水晶，正伸出小爪子，小心翼翼地触摸着水晶的表面。
3. 它坐了下来，抬头凝望着远方的地球，头盔面倒映出地球的影子，眼神里充满了思索和宁静。
4. 它从背包里拿出一面小小的旗子，上面画着一个猫爪印，然后用力地把它插进月球的土壤里，显得非常自豪。

旁白文案：
在遥远的月球上，迎来了一位勇敢的小小探险家。它对这个寂静的世界充满了好奇，每一个角落都值得探索。当它抬头望向那颗蓝色的家园时，小小的身体里涌起了大大的梦想。最终，它留下了属于自己的独特印记，一个代表着勇气和梦想的猫爪。

背景音乐：
/Users/mac/Downloads/penguinmusic-space-chillout-14194.mp3
```
*(注意：背景音乐必须是你电脑上真实存在的绝对路径)*

### 2. 执行生成
在您的 OpenClaw 对话框中，只需把上面这段准备好的**“完整剧本”**发给我，我就会立刻开始全自动在后台为您创作。

*(如果您想脱离 OpenClaw 在终端里独立运行，只需执行 `./scripts/main.sh`，然后把剧本粘贴进去，按 `Ctrl+D` 结束输入即可开始。)*

## ❓ 第三步：详细解答 (FAQ)

**Q1: 为什么我生成的视频是紫色/绿色的纯色图片，且没有声音？**
**A1:** 这是触发了安全保护机制的“模拟模式”。请检查 `run.py` 中的 `OPENAI_API_KEY` 是否已经正确填入了你自己的真实密钥，且不要有前后的空格。

**Q2: 程序报错提示 `ffmpeg: command not found` 怎么办？**
**A2:** 这说明你的电脑系统里没有安装 FFmpeg，或者安装了但没有加入系统环境变量。程序的视频剪辑和混音强依赖此工具。Mac 用户请打开终端运行 `brew install ffmpeg`。

**Q3: 报错提示 `FileNotFoundError: BGM file not found`？**
**A3:** 剧本最后一项提供的“背景音乐”路径不对。请确认那首 MP3 文件确实存在于那个文件夹下。建议在 Mac 上选中音乐文件，按 `Option + Command + C` 复制绝对路径，然后粘贴到剧本里。

**Q4: 我可以生成 10 个镜头的长视频吗？**
**A4:** 当然可以！你只需要在剧本的“镜头描述”部分，按照 `5.`、`6.` 依次往下写即可。程序会自动识别你有多少个镜头，并循环生成。但请注意，镜头越多，消耗的 API Token 和处理时间也会成正比增加。
