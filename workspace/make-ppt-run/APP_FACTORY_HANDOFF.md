Feature: `App Factory` / `Word -> PPT` website with `ppt-master` integration

## 1. Goal of this feature

This feature is a website-style product entry inside `openclaw-control-center`.

Target workflow:

1. User uploads a Word/PDF file
2. User selects a template style
3. System creates a real `ppt-master` project
4. System prepares design spec and source materials
5. System generates SVG pages
6. System exports editable PPTX and SVG-reference PPTX

The final product direction is:

- upload Word
- choose template
- auto-generate premium PPT
- export editable `.pptx`

## 5. Current backend pipeline in detail

When `POST /api/projects` is called, the backend currently does this:

### Step A — create local project folder
Under:
- `openclaw-control-center/runtime/app-factory/projects/<slug-id>/`

Creates:
- `sources/`
- `exports/`
- `outline.md`
- `design-spec.json`
- placeholder export files

### Step B — create real ppt-master project
Runs:
- `project_manager.py init <name> --format ppt169 --dir <runtime/app-factory/ppt-master-projects>`

This creates a real `ppt-master` project directory like:
- `<name>_ppt169_YYYYMMDD`

### Step C — import source
Runs:
- `project_manager.py import-sources <pptProjectPath> <sourcePath>`

### Step D — auto-write design spec
Backend generates and writes:
- `<pptProjectPath>/design_spec.md`

This is currently generated by `buildDesignSpecMarkdown()` inside:
- `openclaw-control-center/scripts/app-factory-preview.cjs`

### Step E — copy selected template assets
Template assets are copied from:
- `ppt-master/skills/ppt-master/templates/layouts/<templateKey>/`

to the generated ppt-master project.

### Step F — try docx to markdown conversion
Runs:
- `doc_to_md.py <source.docx> -o <source.md>`

Important note:
- This only works when the uploaded file is a real Word `.docx`
- During testing, fake `.docx` content was used, so this step failed normally

### Step G — validate project structure
Runs:
- `project_manager.py validate <pptProjectPath>`

### Step H — generate MVP SVG pages
Runs local generator:
- `node openclaw-control-center/scripts/generate-mvp-svg.cjs`

This writes 4 SVG pages into:
- `<pptProjectPath>/svg_final/`

Generated pages:
- `01_cover.svg`
- `02_toc.svg`
- `03_content.svg`
- `04_ending.svg`

### Step I — export final PPTX
Runs:
- `svg_to_pptx.py <pptProjectPath> -s final`

This now successfully exports:
- editable native `.pptx`
- `_svg.pptx` reference file

## 8. What is actually working right now

### Works now
- website UI exists and is usable
- backend API exists and is usable
- project records are created
- real ppt-master project is created
- design spec is auto-written
- template assets are copied
- MVP SVG pages are generated
- final editable PPTX export works

### Important limitation
The current generated PPT content is still **MVP-quality**, not yet real document-driven premium output.

Why?
Because the SVG pages are generated by a local fixed-layout generator, not yet by deep semantic parsing of the uploaded Word content.

So the system is already:
- operational
- end-to-end runnable
- exporting real PPTX

But not yet:
- deeply content-aware
- slide-count adaptive
- premium consulting-grade from arbitrary Word input
