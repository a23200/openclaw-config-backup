- **CRITICAL AUTOMATION FIXES (Douyin Playwright UI Issues)**:
  1.  **The "Enter" Key Trap**: Simulating the `Enter` key inside Douyin's `.zone-container` editor causes the page to abruptly save/exit. **Solution**: Strictly use `locator.fill()` for text input to bypass keyboard event listeners.
  2.  **The Double-Popup Cover Trap**: Setting a 3:4 vertical cover triggers two modals in sequence. 
      - First, the main cropping popup (requires clicking "完成" or "确定"). 
      - Second, a malicious promotional popup pushing horizontal covers. **The script MUST locate and click "暂不设置" (Skip)**, otherwise the entire screen is blocked and publishing fails.
  3.  **Unclickable Elements**: When standard locators fail on Douyin's nested `div` structures, using `.boundingBox()` and clicking relative coordinates (e.g., `box.y - 60` above the text) is a highly effective fallback.
  4.  **Process Safety**: Never use global `pkill -f "Google Chrome"` to clean up Playwright zombies, as it forcibly closes the user's personal browser. Always target the specific Node script process instead.
