
### Xiaohongshu Content Harvester Skill Creation

- **Objective**: Create a skill to scrape post content (titles, authors, URLs, and eventually images/text) from Xiaohongshu's explore feed.

- **Initial Challenge & Solution**: The initial scraping attempts were blocked by Xiaohongshu's anti-bot security which flagged the server IP.
    - **Learning**: Successfully bypassed this by using `agent-browser` with a persistent profile (`--profile`) that was primed by a one-time manual, headed login by the user. This is a critical technique for sites with robust bot detection.

- **URL Discovery Failure**: The primary goal of automatically discovering post URLs from the explore feed ultimately failed due to tooling limitations.
    - **Method**: The `agent-browser snapshot` command could successfully capture a list of on-screen elements (titles, authors).
    - **Root Cause of Failure**: The snapshot does not contain `href` attributes (URLs). All subsequent attempts to query the live browser for the `href` attribute of a captured element (`get attr href @ref`) failed reliably in any scripted context (Node.js or shell). This appears to be a bug or limitation in how `agent-browser` maintains session context for element references across commands.

- **Scripting Iteration & Bugs**: The attempt to create a robust script was fraught with errors, which served as valuable learning experiences.
    - Incorrect session handling in Node.js.
    - Multiple shell scripting bugs, including incorrect quoting (single vs. double), flawed loop logic for multi-line variables, and fragile text parsing.

- **Current Status & Pivot**:
    - The skill, as it stands, can reliably produce a list of **titles and authors** from the explore page, but not their URLs.
    - The user pivoted the task to providing a specific post URL manually for content extraction.
    - The first attempt to navigate to the user-provided detail URL failed; the browser was redirected to the explore feed. This is the immediate problem to be solved next.
