# 2026-03-24 机械臂控制与视觉抓取攻坚战

## 今日成果总结 (老板亲自带队指导)

### 1. 底层控制的避坑与真理
*   **坑：`send_coords` 多轴直线插补的畸变。** 试图让机械臂在保持高度的同时平移，或者全坐标下发时，老版本固件会发生严重的“掉高”（比如前伸时 Z 轴下坠 7 厘米）。底层 IK 解算极其不稳定。
*   **真理：`send_coord` 单轴绝对锁死。** 这是今天的最大收获！如果想绝对笔直下降（只改 Z），或者绝对原地旋转手腕（只改 Rx/Ry），必须调用 `send_coord(id, val, speed)`。它能物理死锁其他 5 个自由度，靠自身代偿完美走出一条直线或完成定点自转。
    *   `id=3`: Z轴（控制垂直高度，如砸下去 `z_drop_30.py`）
    *   `id=4`: Rx（控制横滚角，如手腕翻转 `rx_to_minus_10.py`）
    *   `id=5`: Ry（控制俯仰/偏航，如 `ry_to_90.py`）

### 2. 奇异点 (Singularity Lock) 的破解
*   当机械臂像电线杆一样完全伸直（Z轴到达绝对顶峰 ~421mm）时，所有关节被拉伸到极限。
*   此时如果强制要求改变末端姿态（例如让 Rx 横滚 50 度），底层会因为没有关节弯曲余量而报错拒绝执行。
*   **破解方案：** “先蹲后转”。先将 Z 轴降低 100mm 左右，让肘部（J3）弯曲腾出物理空间，然后再执行极端的姿态翻滚指令 (`escape_singularity_and_roll.py`)。

### 3. “傻瓜式”拖动示教模式
*   如果不想碰坐标系，开发了 `teach_and_play.py`。
*   原理：发送放松指令断电变软 -> 人工摆姿势 -> 记录各关节角度帧 -> 一键通电按顺序安全回放。

### 4. 视觉抓取逻辑 (闭环与降维)
*   放弃了盲目的开环坐标叠加。
*   测试了基于比例尺的降维打击方案：通过全自动平移拍三张照片（`auto_calibrate.py`）得出桌面像素比（1mm=1.5px），然后拍一张照片算出偏移像素，直接转换成毫米发送绝对坐标。
