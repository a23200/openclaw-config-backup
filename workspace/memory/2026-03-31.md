### 机械臂 VLM 控制新架构 (2026-03-31)
- **参考与位置**：学习 `TommyZihao/vlm_arm` 经验，在 `workspace/robot_vlm/` 新建了完全隔离的控制堆栈（附带 `.venv`）。
- **核心机制**：
  1. LLM 充当 Planner，解析自然语言输出带 `back_zero`、`vlm_move` 等可用函数的 JSON 序列。
  2. 视觉大模型（VLM）零样本找物体坐标，返回像素级目标边界框。
  3. 极简两点线性插值标定（手眼映射），将像素直接转为机械臂平面物理坐标。
- **关键硬件细节与偏置处理**：
  - 真实环境为眼在手上（Eye-in-hand），摄像头与末端执行器存在 5cm 横向物理偏置。
  - **策略**：每次拍摄前强制回到旧版的固定 `HOME_ANGLES` `[1.14, 0.43, -4.83, -76.02, 7.55, 133.24]`。通过固定拍摄高度和位姿，把 5cm 的物理距离变成一个“像素常量”，在后续双点标定时直接被 `numpy.interp()` 线性插值吸收掉，完美绕开复杂的数学坐标系换算。

### 手眼标定算法修复 (2026-03-31)
- **踩坑**：测试发现机械臂坐标指反。原因为 `numpy.interp` 要求基准点数组必须是严格递增的，当遇到机械臂/相机坐标系存在镜像或反向映射时，插值会出错导致方向跑偏。
- **修复**：在 `robot_vlm/executor.py` 的 `eye2hand` 方法中废弃了 `numpy.interp`，改写为底层的“基础线性比例映射公式”：计算 `ratio` 后利用 `p1 + ratio * (p2 - p1)` 直接求解。此方法自动免疫轴系反向问题。
- **下一步**：当前 config 中的参考坐标仍是硬编码的 mock 数据，已通知用户利用桌面实物（鼠标、烟盒）跑 `calibrate_helper.py` 获取真实的两点物理与像素坐标，准备完成最终校准。

### 手眼标定与VLM抓取系统迭代 (2026-03-31 晚)
- **二维标定升级为三点仿射变换**: 简单的两点线性插值（甚至比例映射）无法解决摄像机倾斜安装带来的 XY 轴耦合偏差（即物理上一条直线，照片里是斜线）。引入了 OpenCV 的 `cv2.getAffineTransform`，通过 3 个非共线点彻底解决旋转、镜像和缩放畸变。
- **VLM 零样本视觉（Vision Grounding）调优**: VLM 经常因为缺乏绝对尺度概念而返回归一化坐标或“张冠李戴”（把黑色鼠标当成黑色烟盒）。
  - **解决方案**: 在系统 Prompt 中明确告知输入的图片分辨率（例如 1920x1080），并强制约束 X、Y 的取值范围，严禁使用归一化坐标。这能极大提升 VLM 目标检测的空间准确率。
- **机械臂软件保护机制**: 加入了工作台空间限制 (`x_min, x_max, y_min, y_max`)，遇到目标像素换算出界的情况，底层 `safety.py` 会静默拦截并强制机械臂回 Home，防止冲撞损坏。

### 机械臂视觉指向排障补充 (2026-03-31 夜)
- **关键结论**: 只要 `Home` 姿态一变，原有 `calibration_3pt.json` 基本立即失效。因为摄像头高度、旋转、视场都变了，必须重新拍标定图并重做三点映射。
- **关键教训**: 不能依赖 VLM 去“猜”标定图里的物体中心像素来做高精度手眼标定。VLM 适合做零样本目标定位，但不适合承担毫米级标定点采样；其像素误差会直接把仿射矩阵带偏，导致机械臂方向整体错误。
- **正确流程**: 新 Home -> `calibrate_helper_v2.py` 记录 3 个物理点 -> `click_calib.py` 人工点击同一接触点的像素坐标 -> 固化 `calibration_3pt.json` -> 再做 VLM 指向测试。
